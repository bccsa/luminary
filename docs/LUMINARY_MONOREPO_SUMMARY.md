# Luminary Monorepo Summary (main branch)

Luminary is an offline-first content platform (API, app, CMS, shared library). Auth is JWT-based (e.g. Auth0). The API validates the token and sends a **clientConfig** over Socket.IO with an **access map** and upload limits.

**Permission system.** The access map is group ID → DocType → AclPermission (view, edit, delete, assign, translate, publish). The API derives it from the user’s group memberships; the app/CMS store it via the shared library as reactive `accessMap`. `getAccessibleGroups(permission)` returns which groups the user can access per DocType; `verifyAccess` and `hasAnyPermission` gate UI and sync. Only data for groups and doc types the user has at least View on is synced or returned.

**Syncing.** Clients use the shared library. They POST to the REST **query** endpoint with a selector (type, parentType for content) and optional `gapStart`/`gapEnd`. The API returns document chunks by `updatedTimeUtc`. The client keeps a **SyncMap** in IndexedDB per doc type and group and requests further chunks to fill gaps. Socket.IO pushes live **data** and **clientConfig**. The app syncs Language first, then Posts, Tags, and their Content, using `getAccessibleGroups(View)` so only permitted groups are included.

**Query.** The query service accepts a Mango-style selector, expands it with permission checks, and enforces View access for the requested type and memberOf groups. Results include optional `blockStart`/`blockEnd`. Doc types on main: Content, Group, Language, Redirect, Post, Tag, User, DeleteCmd, Storage, Crypto. Content is language-scoped; sync and socket filtering respect optional app language lists.

---

**Main files and what gets the system working**

- **API:** `api/src/main.ts` — bootstrap: loads design docs (`db.seedingFunctions.upsertDesignDocs`), optional seed, `PermissionSystem.init(dbService)`, schema upgrade, CORS. `api/src/app.module.ts` — wires controllers (Query, Search, ChangeRequest, StorageStatus) and providers (DbService, Socketio, QueryService, etc.). `api/src/socketio.ts` — WebSocket gateway: auth, `joinSocketGroups`, then emits **clientConfig** (access map + upload limits) and **data** (doc chunks). `api/src/endpoints/query.controller.ts` + `query.service.ts` — POST `/query`: validate Mango body, resolve JWT, enforce permissions, call `db.executeFindQuery`. `api/src/db/db.service.ts` — CouchDB (nano) wrapper: `executeFindQuery`, getDocs, design-doc indexes. `api/src/permissions/permissions.service.ts` — `PermissionSystem.init()` loads groups and builds the access map from group ACLs; used by query and socket to compute what to send. `api/src/jwt/processJwt.ts` — validates JWT and returns user details for permission lookups.
- **Shared:** `shared/src/luminary.ts` — `init(config)`: sets config, `initDatabase()`, `getSocket()`, `getRest()`, and `initSync(http)` so the client can sync. `shared/src/db/database.ts` — Dexie/IndexedDB schema, SyncMap storage, `bulkPut` for incoming docs. `shared/src/socket/socketio.ts` — Socket.IO client: connect with token, emit `joinSocketGroups`, on **data** persist to DB and on **clientConfig** update `accessMap` and `isConnected`. `shared/src/rest/http.ts` — `HttpReq`: authenticated POST to API (used by sync). `shared/src/rest/sync2/sync.ts` — `sync(options)` and `initSync(http)`: gap-based REST sync, SyncMap updates, merge/trim. `shared/src/permissions/permissions.ts` — `accessMap` ref, `getAccessibleGroups`, `verifyAccess`, `hasAnyPermission`.
- **App:** `app/src/main.ts` — creates Vue app, calls `init()` from luminary-shared with `apiUrl`, `syncList`, and token (from auth); then `initLanguageSync()` and `initSync()`. `app/src/sync.ts` — watchers on `accessMap` / `isConnected` / `appLanguageIdsAsRef` bump sync iterators; `initLanguageSync()` syncs Language when connected; `initSync()` syncs Posts, Tags, and their Content for accessible groups. `app/src/auth.ts` — Auth0 (or equivalent) setup and token supply for `init()`. Together, main → init (DB + socket + REST + sync) and sync.ts → sync runs drive the offline-first loop.
